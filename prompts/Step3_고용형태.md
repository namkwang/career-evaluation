### Phase 0: 전반적 고용 성향 파악 (가장 먼저 수행)

개별 경력을 판정하기 전에, 이 지원자의 전체 경력 흐름을 먼저 파악하세요.

핵심 판단 기준:
- 종합건설 상위권(ranking_position이 50 이하)에서 장기 정규직 근무 이력이 있는 사람은 이후 경력도 정규직일 가능성이 매우 높음
- 건설업에서 정규직→비정규직으로 전환하는 경우는 극히 드묾 (예외: 정년 퇴직 후 감리/전문업체 재취업, 임원 퇴임 후 재취업)
- 반대로, 초기 경력부터 1사 1프로젝트 패턴이면 이후 애매한 경력도 비정규직일 가능성이 높음

판단 방법:
1. [코드 사전 분석 결과]에서 회사별 판정을 전체적으로 조망
2. "정규직 가능성 높음"이 주를 이루면 → 정규직 성향
3. "비정규직 가능성 높음"이 주를 이루면 → 비정규직 성향
4. 이 판단을 Phase A, B에서 애매한 경력 판정 시 참고

### Phase A: 전체 이력 패턴 분석 (Phase 0 이후 수행)

Phase 0에서 파악한 전반적 성향을 참고하며, 경력 이력 전체를 조망하여 판정합니다.

**A-1. 서류 명시 확정:**
각 경력에서 서류에 명시적 표현이 있는 경우 즉시 확정합니다.
- position_raw에 "전문직", "촉탁", "계약직", "현채" 등이 포함 → contract 확정
- 이력서에 "계약직", "전문직", "촉탁", "파견", "위촉" 등이 명시 → contract 확정
- 서류에 "정규직", "일반직" 등이 명시 → regular 확정

**A-2. 현채직(현장채용) 패턴 감지:**
A-1에서 확정되지 않은 경력들의 전체 이력 패턴을 분석합니다.
입력 데이터에 [코드 사전 분석 결과]가 포함되어 있으므로 반드시 참고하세요.

다음 조건 중 하나에 해당하면 **현채직 패턴**으로 판정합니다:

**(패턴 1) 서로 다른 법인에서 각각 1건만 수행:**
- 서로 다른 법인(회사)에서 각각 프로젝트 1건만 수행하고 퇴사하는 패턴이 2회 이상 반복

**(패턴 2) 같은 회사에서 work_history가 끊어져 반복 등장:**
- 경력증명서의 work_history(근무처)에서 같은 회사가 2건 이상으로 끊어져 등장하는 경우
- 이는 고용이 끊어졌다 재개된 것 = 비정규직(현채직)
- [코드 사전 분석 결과]에서 "work_history가 끊어져 반복 등장" 또는 "비정규직 가능성 높음"으로 표시된 회사를 확인하세요
- 해당 회사의 모든 경력을 contract로 판정

**(패턴 2 예외) 같은 회사 + work_history 1건 연속 + 프로젝트 간 공백:**
- work_history가 하나로 이어지는데 프로젝트 사이에 공백이 있는 경우
- 이는 정규직이 현장 A 끝나고 다음 현장 B 배치 전 본사 대기하는 기간
- [코드 사전 분석 결과]에서 "정규직 가능성 높음: work_history 1건 연속"으로 표시된 회사
- 이 경우는 비정규직으로 판정하지 마세요

현채직 패턴이 감지되면:
- **해당 패턴에 속하는 모든 경력을 빠짐없이 contract로 확정합니다**
- 일부만 contract로 판정하고 나머지를 다른 것으로 남기지 마세요

**Phase A 완료 후, 확정된 경력 목록을 기록하세요. Phase B에서 이 경력들의 판정을 변경하면 안 됩니다.**

### Phase B: 개별 경력 분석 (Phase A에서 미확정된 것만)

Phase A에서 확정되지 않은 경력만 대상으로 아래를 수행합니다.

**B-1. 개별 신호 분석:**
각 미확정 경력에 대해 아래 신호를 확인합니다. 2개 이상 해당하면 contract, 1개면 unknown + flag:
- (a) 한 회사에서 경력 레코드 1건만 수행하고 퇴사
- (b) 같은 회사를 퇴사 후 재입사하는 패턴 (동일 법인 반복 등장)
- (c) 경력증명서에서 동일 회사 근무처가 짧게 끊어져 반복 등장
- (d) 동일 회사 개별 근무기간이 3년 미만

**B-2. 정규직 판정:**
B-1에서도 확정되지 않은 경력에 대해, 다음 조건을 **모두** 충족하는 경우에만 regular로 확정:
1. 한 회사에서 별도 기간으로 구분된 경력 레코드 2건 이상
2. [코드 사전 분석 결과]에서 해당 회사가 "정규직 가능성 높음"으로 표시 (work_history 1건 연속)

하나라도 충족하지 못하면 B-2를 적용하지 마세요.
- work_history가 끊어져 반복 등장하면 절대 정규직으로 보지 않습니다
- 하나의 레코드에 콤마로 나열된 현장은 별도 프로젝트가 아닙니다

**B-3. 미확인:**
B-1, B-2 어디에도 해당하지 않는 경우 → unknown + verification_flag: "contract_status_uncertain"

## 출력 JSON 스키마

    }

## 주의사항

1. **역할 범위 엄수**: 고용형태 판정만 수행합니다. company_category, is_small_company, ranking_year, ranking_position, working_days 등은 입력값 그대로 출력하세요.

2. **패턴 판정(②)에서는 경력 이력 전체를 조망하세요**: 개별 경력만 보지 말고, 전체 이직 패턴을 함께 고려합니다.

3. **신호 2개 이상이 동반되어야 contract로 판정합니다**: (a)~(d) 중 1개만 해당하면 unknown + flag로 처리하세요. 명시적 표현이 없고 패턴 신호도 없는 경우는 무조건 unknown입니다 (regular로 임의 확정하지 마세요).

4. **employment_type_reason은 상세하게 기록하세요**: 어떤 기준(①~④)의 어떤 항목에 해당하는지, 구체적 근거(position_raw 원문, 이직 패턴 설명 등)를 포함합니다.

5. **현장채용 패턴(e)은 가장 강한 패턴 신호입니다**: 여러 법인에서 각각 1건만 수행하고 퇴사하는 패턴이 2회 이상 반복되면, 개별 신호(a~d) 개수와 관계없이 즉시 contract로 판정합니다.
