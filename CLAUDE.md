# 경력산정 자동화 웹앱 프로젝트 계획서

## 1. 프로젝트 개요

건설회사 채용 시 지원자의 외부 경력을 자동 산정하는 웹앱.
지원자가 제출한 이력서와 건설기술인협회 경력증명서를 AI로 분석하여 경력연차를 산출한다.

사용자는 건설회사 인사/채용 담당자이며, 핵심 흐름은 다음과 같다:
```
서류 업로드 → AI 추출 → AI 병합/회사확정 → 담당자 확인 → AI 경력산정 → 결과 출력
```

---

## 2. 처리 파이프라인

| Step | 이름 | 처리 주체 | 프롬프트 | 비고 |
|---|---|---|---|---|
| 1-A | 경력증명서 추출 | AI (Vision) | prompts/01_경력증명서_추출.md | |
| 1-B | 이력서 추출 | AI (Vision) | prompts/02_이력서_추출.md | |
| 1-C | 도급순위 매칭 | 코드 (백엔드) | - | 엑셀 DB에서 필터링 |
| 2 | 데이터 준비 + 회사 확정 | AI + 웹검색 | prompts/03_경력산정_Step2.md | |
| - | 담당자 확인 | 사람 (UI) | - | 플래그 항목 수정 |
| 3 | 인정률 + 경력산정 | AI | prompts/04_경력산정_Step3.md | |

각 Step의 비즈니스 로직, 입출력 JSON 스키마, 주의사항은 해당 프롬프트 파일에 상세 기술되어 있다.

### Step 1-C: 도급순위 필터링 (코드 처리)
- 도급순위 엑셀: 약 30년치, 200위까지, 4000행 이상
- Step 1에서 추출된 회사명 + 근무연도 목록을 코드로 추출
- 해당 연도의 200위 리스트를 엑셀에서 필터링하여 Step 2 입력으로 제공
- 사명 변경/합병 등의 퍼지 매칭은 코드가 아닌 Step 2 AI가 처리

## 4. UI/UX 흐름

### 화면 1: 서류 업로드
- 이력서 PDF 업로드 (필수)
- 경력증명서 PDF 업로드 (선택)
- 지원 직무 선택 (건축/토목)
- 채용 유형 선택 (일반/전문직/현채직)

### 화면 2: 추출 결과 확인
- Step 1 추출 결과를 테이블로 표시
- 원문 PDF와 나란히 대조 가능
- 추출 오류 수동 수정 가능

### 화면 3: 경력 병합 + 회사 정보 확인 (핵심 화면)
- Step 2 결과를 편집 가능한 테이블로 표시
- 편집 항목: 회사 유형(드롭다운), 10인 미만 여부(체크박스), 고용형태(드롭다운)
- verification_flag 항목 하이라이트 + 설명
- 연속근무군 시각적 그룹 표시
- 담당자 수정 후 [확정]

### 화면 4: 경력산정 결과
- 최종 경력연차 표시
- 경력 상세: 구간별 인정률, 인정일수
- 인정률별 요약
- 학력 차감 내역
- [PDF/엑셀 내보내기]

## 5. 데이터

### 도급순위 엑셀
- 약 30년치, 200위까지, 4000행 이상
- 앱 초기화 시 DB 또는 메모리에 로드
- 컬럼 구조는 엑셀 파일 확인 후 확정

### 지원자 처리 결과
- 각 Step별 JSON 저장
- PDF 원본 보관

## 6. 개발 우선순위

### Phase 1: MVP
1. 도급순위 엑셀 로드 + 필터링
2. Step 1-A/1-B API 호출 + 결과 표시
3. Step 2 API 호출 + 담당자 확인 UI
4. Step 3 API 호출 + 결과 표시

### Phase 2: 사용성
5. PDF 뷰어 (원문 대조)
6. 추출 결과 수동 수정 UI
7. 결과 내보내기 (PDF/엑셀)
8. 지원자별 이력 저장

### Phase 3: 고도화
9. 배치 처리 (다수 지원자 일괄)
10. Step 3 계산 코드 검증 (AI 결과 재검산)
11. 통계/대시보드

---

## 7. 주의사항

- **경력증명서 미제출**: 이력서만으로 처리 가능해야 함
- **PDF 품질**: 스캔/사진 등 저품질 대응, 추출 실패 시 수동 입력 대안
- **계산 정확도**: AI 계산 오류 가능성 있으므로 Phase 3에서 코드 재검산 추가 권장
- **개인정보**: 주민등록번호 뒷자리 마스킹, PDF 보관 정책 결정 필요
