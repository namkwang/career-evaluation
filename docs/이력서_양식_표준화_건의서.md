# 이력서 양식 표준화 건의서

## 1. 배경

경력산정 자동화 시스템 MVP 개발·테스트 과정에서, 현재의 자유양식 이력서로는 정확한 경력산정이 구조적으로 어렵다는 점이 반복적으로 확인되었습니다. 이 문서는 테스트에서 발견된 실제 사례를 바탕으로, 이력서 양식 표준화의 필요성과 기대 효과를 정리합니다.

---

## 2. 현재 문제점

### 2-1. 고용형태 판단 불가

**현상**: 자유양식 이력서에는 고용형태(정규직/계약직/현채직) 기재란이 없어, AI가 경력 패턴을 분석하여 추측해야 합니다.

**실제 발생한 오류 사례:**

| 사례 | AI 판정 | 실제 | 원인 |
|---|---|---|---|
| 포스코건설 11년 근무, 6개 현장 | 정규직 | 비정규직(현채직) | 장기근속+직급상승으로 정규직 오판 |
| 다원조경 2회 근무 | 정규직 | 비정규직 | "프로젝트 2건 이상"을 정규직 기준으로 오판 |
| 여러 조경업체 전전 | 일부 정규직/일부 미확인 | 전부 현채직 | 현채직 패턴 일부만 감지 |

**근본 원인**: 경력증명서에도 고용형태 필드가 없으며, 직위(position)에 "전문직차장" 등으로 간접 기재된 경우만 판별 가능합니다. 명시적 표기가 없는 대다수의 경력은 AI가 추측할 수밖에 없고, 이 추측은 자주 틀립니다.

### 2-2. 연속근무(법인 이동) 판단 불가

**현상**: 동일 그룹사 내 법인 이동(공동도급사 파견, 계열사 전출 등)은 인정률 100%를 적용해야 하지만, 이력서에 그룹사 관계나 이동 사유가 기재되지 않습니다.

**실제 발생한 오류 사례:**

| 사례 | AI 판정 | 문제점 |
|---|---|---|
| 재운종합건설 → 대림산업 | 연속근무(같은 그룹) | 실제로는 무관한 회사 — AI가 날짜 연속만으로 묶음 |
| 안산조경 → 다원조경 → 에스에이치조경 | 전부 연속근무 | 현채직이 현장을 옮긴 것인데 법인이동으로 오판 |
| 남광토건 → 극동건설 | 판단 불가 | 실제 같은 그룹(남광그룹)이나 이름으로 판별 불가 |

**근본 원인**: 법인 이동 여부는 지원자 본인만 알 수 있는 정보입니다. 회사명만으로 그룹사 관계를 판단하는 것은 한계가 있으며, AI 웹검색으로도 공동도급 관계나 비공개 계열사는 확인이 어렵습니다.

### 2-3. 회사 유형 판단 오류

**현상**: 도급순위에 없는 회사는 AI가 웹검색으로 업종을 판단하는데, 전문건설/건설유관/기타 분류가 부정확한 경우가 있습니다.

**실제 발생한 사례:**
- 신원씨케이(전문건설업체)를 종합 100위 이내로 분류
- 대아건설을 과거 순위(2001년 37위) 기준으로 100위 이내 분류 (현재 131위)

### 2-4. AI 판단의 일관성 부족

동일한 이력서를 여러 번 분석해도 결과가 달라지는 경우가 있습니다. 프롬프트를 아무리 상세하게 작성해도 AI가 규칙을 무시하거나, 텍스트 설명과 코드값이 불일치하는 문제가 반복적으로 발생합니다.

| 현상 | 빈도 |
|---|---|
| 이전 단계 확정값을 다음 단계에서 임의 변경 | 매우 빈번 |
| 고용형태 판정 결과가 실행마다 다름 | 간헐 |
| company_category_reason과 company_category 불일치 | 빈번 |

## 3. 이력서 양식 표준화 제안

### 3-1. 필수 기재 항목 추가

현재 이력서에 없거나 선택적인 항목 중, 경력산정에 필수적인 항목을 표준 양식에 포함합니다.

| 항목 | 형식 | 용도 |
|---|---|---|
| **고용형태** | 체크박스: 정규직 / 계약직 / 파견직 / 현장채용 | AI 추측 제거 → 검증만 수행 |
| **이전 회사와의 관계** | 체크박스: 해당없음 / 동일그룹 계열사 / 공동도급사 파견 / 사명변경 | 연속근무 판단 근거 |
| **퇴직사유** | 선택형: 계약만료 / 자발적 이직 / 권고사직 / 법인이동 / 기타 | 고용형태 교차 검증 |

### 3-2. 경력 기재 구조 표준화

| 항목 | 현재 | 제안 |
|---|---|---|
| 회사명 | 자유 기재 | 정식 법인명 기재 (사업자등록증 기준) |
| 근무기간 | YYYY-MM 또는 YYYY | **YYYY-MM-DD ~ YYYY-MM-DD** 필수 |
| 현장/프로젝트 | 기재 안 하는 경우 많음 | 현장명 + 기간 별도 기재 |
| 직위 | 자유 기재 | 선택형: 사원/주임/대리/과장/차장/부장/이사 + 접두어(전문직/촉탁 등) |

### 3-3. 양식 예시

```
[경력사항]

회사명: _________________ (정식 법인명)
근무기간: ____년 __월 __일 ~ ____년 __월 __일
고용형태: □ 정규직  □ 계약직  □ 파견직  □ 현장채용
직위: □ 전문직 접두어 해당  직급: ________
퇴직사유: □ 계약만료  □ 자발적이직  □ 법인이동  □ 기타(_______)
이전 회사와의 관계: □ 해당없음  □ 동일그룹 계열사  □ 공동도급사 파견  □ 사명변경

  [현장/프로젝트]
  1. 현장명: _________ 기간: ____년__월__일 ~ ____년__월__일
  2. 현장명: _________ 기간: ____년__월__일 ~ ____년__월__일
```

## 4. 기대 효과

### 4-1. 경력산정 정확도 향상

| 항목 | 현재 (자유양식) | 표준화 후 |
|---|---|---|
| 고용형태 판정 | AI 패턴 추측 (오류 빈번) | 지원자 기재 + AI 검증 |
| 연속근무 판정 | AI 웹검색 추측 (한계) | 지원자 기재 + 경력증명서 대조 |
| 근무일수 계산 | 월 단위 추정 (오차 ±30일) | 일 단위 정확 계산 |
| 회사 정보 확인 | AI 웹검색 (오류 가능) | 정식 법인명으로 정확 매칭 |

### 4-2. 업무 효율 향상

| 항목 | 현재 | 표준화 후 |
|---|---|---|
| AI 처리 단계 | 5단계 (추출→병합→고용형태→산정) | 3단계로 축소 가능 (추출→병합+산정) |
| 담당자 검토 시간 | 매 건 30분+ (오류 수정) | 10분 이내 (확인만) |
| AI 비용 | 4회 API 호출 | 2회로 축소 가능 |

### 4-3. AI 역할 변화

| 현재 | 표준화 후 |
|---|---|
| 추측: 고용형태를 패턴으로 추측 | 검증: 지원자 기재와 경력증명서 대조 |
| 추측: 그룹사 관계를 웹검색으로 추측 | 검증: 지원자 기재 + 사업자 정보 대조 |
| 추측: 퇴직사유를 유추 | 확인: 기재된 퇴직사유 확인만 |

---

## 5. 결론

현재 자유양식 이력서로는 AI가 아무리 정교하게 분석해도 **지원자만 아는 정보**(고용형태, 법인이동 사유)를 정확히 판단할 수 없습니다. 이는 AI의 한계가 아니라 **입력 데이터의 한계**입니다.

이력서 양식 표준화를 통해:
1. 경력산정 정확도를 근본적으로 개선하고
2. 담당자 업무 부담을 줄이며
3. AI 처리 비용과 시간을 절감할 수 있습니다.
